# VC-Framework (Vibe Coding Framework)

> 인간이 정의하고, AI가 실행하고, 인간이 검증한다.

**VC-Framework**는 소프트웨어 개발 생명주기(SDLC)를 인간과 AI의 협업 구조로 재설계한 방법론입니다.

🌐 **공식 소개 페이지**: [https://vc-framework.com](https://vc-framework.com)

---

## VC-Framework란?

기존 소프트웨어 개발은 인간 중심으로 설계되었습니다.  
VC-Framework(Vibe Coding Framework)는 **AI를 실행 주체**로, **인간을 정의·검증 주체**로 재정립한 새로운 개발 패러다임입니다.

- **V**ibe — 개발자의 의도와 맥락을 AI에게 전달하는 방식
- **C**oding — AI가 코드를 생성·실행하는 단계
- **Framework** — 이를 체계화한 7단계 협업 구조

---

## 7단계 프레임워크

| 단계 | 이름 | 주체 | 설명 |
|------|------|------|------|
| 110 | **Frame** | 인간 | 목적·범위·제약 정의 |
| 120 | **Scope** | 인간 + AI | 요구사항 구체화 |
| 130 | **Design** | 인간 + AI | 아키텍처·UI 설계 |
| 140 | **Build** | AI | 코드 생성 및 구현 |
| 150 | **Review** | 인간 | 코드 검토 및 피드백 |
| 160 | **Verify** | 인간 + AI | 테스트 및 품질 검증 |
| 170 | **Operate** | 인간 + AI | 배포·운영·모니터링 |

---

## 핵심 개념

### Vibe (바이브)
개발자의 의도, 맥락, 감각을 AI가 이해할 수 있는 형태로 구조화하는 것.  
단순한 프롬프트가 아닌, **AI Execution Context(AEC)** 를 통해 AI에게 전달됩니다.

### AI Execution Context (AEC)
AI가 코드를 생성하기 위해 필요한 모든 맥락 정보의 집합.  
목적, 제약, 환경, 스타일, 참조 코드 등을 포함합니다.

### Human-in-the-Loop
모든 단계에서 인간의 판단이 개입되는 구조.  
AI는 실행하고, 인간은 방향을 결정합니다.

## 기존 개발 방식과의 차이

| 구분 | 기존 SDLC | VC-Framework |
|------|-----------|--------------|
| 코드 작성 | 인간 | AI |
| 방향 결정 | 팀 회의 | 인간 단독 |
| 실행 속도 | 느림 | 빠름 |
| 품질 책임 | 개발자 | 인간 + AI 공동 |
| 반복 주기 | 주 단위 | 시간 단위 |
