# VC-Framework (Vibe Coding Framework)

> 인간이 정의하고, AI가 실행하고, 인간이 검증한다.

**VC-Framework**는 Vibe Coding(바이브 코딩)·AI Coding(에이아이 코딩)을 소프트웨어 개발 전 생애주기에 적용할 수 있도록 체계화한 협업형 소프트웨어 개발 방법론입니다.

🌐 **공식 소개 페이지**: [https://vc-framework.com](https://vc-framework.com)

---

## VC-Framework란?

기존 소프트웨어 개발은 인간 중심으로 설계되었습니다.  
VC-Framework(Vibe Coding Framework)는 **AI를 실행 주체**로, **인간을 정의·검증 주체**로 재정립한 새로운 개발 패러다임입니다.

- **V**ibe — 개발자의 의도와 맥락을 AI에게 전달하는 방식
- **C**oding — AI가 코드를 생성·실행하는 단계
- **Framework** — 이를 체계화한 7단계 협업 구조

---

## Vibe Coding · AI Coding과의 관계

**Vibe Coding**(바이브 코딩)은 개발자가 코드를 직접 작성하는 대신, 의도와 맥락을 AI에게 전달하여 코드를 생성하게 하는 방식입니다.  
**AI Coding**(에이아이 코딩, AI 코딩)은 AI 도구를 활용한 소프트웨어 개발 전반을 의미합니다.

VC-Framework는 이 Vibe Coding·AI Coding 방식을 **소프트웨어 개발 전 생애주기**에 적용할 수 있도록 7단계 구조로 정립한 프레임워크입니다.  
단순한 프롬프트 작성을 넘어, 각 단계별 입력·실행·검증을 체계화합니다.

---

## 7단계 프레임워크

| 단계 | 이름 | 주체 | 설명 |
|------|------|------|------|
| 110 | **Frame** | 인간 + AI | 목적·범위·제약 정의 |
| 120 | **Discover** | AI | AI 기반 서비스 탐색 |
| 130 | **Design** | AI | AI 기반 시스템 설계 |
| 140 | **Build** | AI | AI 기반 코드 구현 |
| 150 | **Validate** | 인간 + AI | 인간 참여 기반 검증 |
| 160 | **Release** | AI | AI 기반 배포 |
| 170 | **Operate** | AI | AI 기반 운영 |

## 핵심 개념

### Vibe (바이브)
개발자의 의도, 맥락, 감각을 AI가 이해할 수 있는 형태로 구조화하는 것.  
단순한 프롬프트가 아닌, **AI Execution Context(AEC)** 를 통해 AI에게 전달됩니다.

### AI Execution Context (AEC)
AI가 코드를 생성하기 위해 필요한 모든 맥락 정보의 집합.  
목적, 제약, 환경, 스타일, 참조 코드 등을 포함합니다.

### Human-in-the-Loop
모든 단계에서 인간의 판단이 개입되는 구조.  
AI는 실행하고, 인간은 방향을 결정합니다.

## 기존 개발 방식과의 차이

| 구분 | 기존 SDLC | VC-Framework |
|------|-----------|--------------|
| 코드 작성 | 인간 | AI |
| 방향 결정 | 팀 회의 | 인간 단독 |
| 실행 속도 | 느림 | 빠름 |
| 품질 책임 | 개발자 | 인간 + AI 공동 |
| 반복 주기 | 주 단위 | 시간 단위 |
